## 1️⃣ 프로젝트 총괄 및 인프라 (devman26173)

> **발표자**: devman26173

저는 프로젝트의 전체 아키텍처 설계와 인프라 구성을 담당했습니다.

**중점을 둔 부분**은 팀원들이 개발에 집중할 수 있도록 환경을 안정적으로 구성하는 것이었습니다.

먼저, Spring Boot 프로젝트를 초기화하고 `pom.xml`에 필요한 의존성을 세팅했습니다.
그리고 Docker와 `docker-compose.yml`을 활용해 로컬 환경에서도 MySQL을 통일된 환경으로 실행할 수 있도록 구성했으며,
AWS EC2 배포를 위한 `application-prod.yml`과 `Dockerfile`도 함께 작성했습니다.

페이지 개발 측면에서는 사용자가 처음 접하는 **홈 페이지**와 게시판 목록 페이지를 제작하고,
서비스 전체에서 공통으로 사용하는 **메뉴바**와 오류 발생 시 보여주는 **에러 페이지**도 개발했습니다.

또한, Git 브랜치 전략을 수립하고 팀원들의 PR을 검토하여 merge하는 역할도 맡았습니다.

## 2️⃣ 사용자 인증 시스템 (goodsujin)

> **발표자**: goodsujin

저는 사용자 **회원가입, 로그인, 로그아웃** 그리고 **비밀번호 관련 기능** 전반을 담당했습니다.

**중점을 둔 부분**은 보안성과 인증 흐름의 정확성이었습니다.

회원가입 시 아이디 중복 체크 로직을 구현했고,
비밀번호는 `BCryptPasswordEncoder`를 사용해 암호화하여 DB에 저장되도록 처리했습니다.
덕분에 실제 비밀번호가 노출되지 않고 안전하게 관리됩니다.

로그인 인증 과정에서는 DB에서 아이디를 조회한 뒤, 입력한 비밀번호와 저장된 암호화된 비밀번호를 비교하여 인증합니다.
디버깅 과정에서 각 단계별 처리 결과를 콘솔에 출력하도록 구현해, 어느 단계에서 오류가 발생하는지 빠르게 파악할 수 있었습니다.

또한 로그인 상태는 세션으로 관리하며, 로그아웃 시 세션을 완전히 종료합니다.
회원 탈퇴와 비밀번호 변경 기능도 구현했으며, 특히 변경 작업은 데이터 정합성을 보장하기 위해 트랜잭션 처리를 적용해 오류 발생 시 자동으로 롤백되도록 했습니다.

## 3️⃣ 게시판(FoodBoard) 시스템 (devhyunju)

> **발표자**: devhyunju

저는 프로젝트의 핵심 기능인 **음식 게시판(FoodBoard)** 전반을 담당했습니다.

**중점을 둔 부분**은 게시판 CRUD를 완성하면서도, 일본이라는 지역적 특성을 살린 **지역 필터링** 기능을 구현하는 것이었습니다.

게시글 작성·수정·삭제·조회의 기본 CRUD 기능을 구현했고, 이미지 파일은 AWS S3와 Cloudflare R2를 통해 업로드되도록 처리했습니다.

**지역 필터링** 기능은 처음에 어떻게 구현할지 고민이 많았는데,
일본의 8개 지방과 각 지방에 속하는 도도부현을 `HashMap`으로 미리 매핑해두고,
사용자가 지방을 선택하면 해당 도도부현 리스트를 꺼내 JPA의 `In` 쿼리로 한 번에 조회하는 방식으로 해결했습니다.

**검색 기능**은 제목이나 내용 중 어느 곳에 키워드가 포함되어도 검색 결과에 나오도록 구현했습니다.

그 외에도 조회수 증가 기능과 공지사항 페이지를 추가로 작성했습니다.

## 4️⃣ 게시글 댓글 및 좋아요 기능 (java0731kk)

> **발표자**: java0731kk

저는 게시글에 달리는 **댓글, 대댓글**, 그리고 **좋아요** 기능을 담당했습니다.

**중점을 둔 부분**은 단순히 기능을 동작시키는 것을 넘어서, 사용자 경험까지 고려한 설계였습니다.

댓글은 `parentId`를 활용한 **자기 참조 구조(중첩 구조)**로 설계해, 댓글에 대한 답글도 달 수 있게 구현했습니다.
댓글 목록은 최신순과 오래된순으로 정렬할 수 있으며, 댓글 편집·삭제는 작성자 본인만 가능하도록 권한 제어를 구현했습니다.

**좋아요 기능**은 좋아요 버튼을 누를 때마다 페이지를 새로 고침하지 않아도 상태가 즉시 반영되도록 **Ajax 비동기 토글 방식**으로 구현했습니다.
DB에 좋아요 레코드가 있으면 삭제(취소), 없으면 추가하는 토글 방식입니다.

**비로그인 접근 처리**도 신경 썼는데, 단순히 오류 메시지를 보여주는 것이 아니라
알림 창을 띄운 뒤 자연스럽게 로그인 페이지로 유도하여 사용자가 불편함 없이 다음 행동을 취할 수 있도록 설계했습니다.

## 5️⃣ 프로필 관리 시스템 (min_chang_isaac)

> **발표자**: min_chang_isaac

저는 **프로필 관리 시스템**과 프로젝트의 **데이터 모델 설계(ERD)**를 담당했습니다.

**중점을 둔 부분**은 비즈니스 로직에 맞는 데이터베이스 관계를 정확하게 설계하고, 사용자 경험 측면에서 자신의 활동을 한눈에 볼 수 있는 프로필 화면을 구현하는 것이었습니다.

ERD 설계 단계에서 User, Profile, Post, Comment, Like, FoodBoard 엔티티 간의 관계를 정의하고, JPA의 `@OneToOne`, `@ManyToOne` 등의 어노테이션으로 관계 매핑을 구현했습니다.

**프로필 기능**은 처음 프로필 페이지에 접근했을 때 데이터가 없으면 자동으로 빈 프로필을 생성해주는 방식으로 구현했습니다.
이를 통해 회원가입 즉시 프로필 페이지를 이용할 수 있습니다.

또한, 프로필 화면에서 해당 사용자가 작성한 게시글과 댓글을 함께 조회할 수 있어,
자신의 활동 내역을 한 곳에서 확인할 수 있습니다.
